HTML

  • HTML,or Hyper Text Markup Language, is the basic buiding block of a website. It handles the content of the website, the structure, and the meaning of each element on the page.
  • The head element contains information, also known as metadata, about the webpage that the browser uses to serve said webpage to the user.
  • This information is known as metadata and is only visible to the developer; other than the title.
  • The title element is extremely important for Search Engine Optimization (SEO).
  • The viewport gives the browser instructions on how to control the pages dimensions and scaling.
  • Elements nested within another element are reffered to as child elements, while the element that is containing, is called the parent element.
  • The body element represents the visible content shown to the user.
  • It is good practice to use semantic elements whenever possible, especially when content will be visible to users.
  • The words element and tag are sometimes used interchangebly but there is a difference.
  • Elements refer to the content from opening tab to closing tab.
  • Tags refer only to what is within the angled brackets.
  • The element br, creates a line of enmpty space or a line break between two blocks of information.
    • The element a, stands for anchor and creates a link to the same or other websites.
  • The style element defines the CSS styles associated with the website.

CSS

  • CSS, or Cascading Style Sheets, is a programming language that defines how webpage content is rendered to the users. It provides styling visual layout options.
  • An inline CSS is used when a developer wants to style and elemend directly in the HTML file. These are useful for quick changes but are permanent and sometimes less favorable than other methods.
  • Internal style sheets are entirely embeded in an HTML file using a style element nested in the head parent element. This contains CSS rules for the webpage. These rules only applly to that page and classes and id's can be used to style multiple elements simultaneously.
  • External CSS style sheets allows developers to keep all their rules in a separate file, which makes design changes easier. This is untimately the preferred method of the three style sheets.
  • SoC or separation of concerns is the principle of separting code into different files based on use. In other words, each section of code should have their own responsibility.
  • A program that utilizes Soc is considered modular. Modules can espose different versions of interface, which increases freedom to upgrade and update a complex system in peice-meal fashion, without interm loss of function.
  • Encapsulating is the storing of information inside a section of code with a well defined interface.
  • The Selector defines the element or attribute to which the rules or declaration will apply.
  • Declarations contain two important components; the CSS propertey we want to applly and the value of the property. (ex: color:blue)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Colors have corresponding hexadecimal codes or hex #'s that can be refrenced.

Git

  • Git is a version control system that alows developers to manage and keep track of each version of their project, locally on your own coputer.
  • GitHub is a cloud-based hosting service that allows developers to keep track of our projects inside repositories that are stored remotely.
  • GitHub also provides the abilitiy for multiple developers to simultaneously work on the same project without affecting each workprocess.
  • Git status: checks as to which branch we are currently operation on
  • Git checkout branch-name: will allow you to navigate to said branch.
  • Git checkout -b branch-name: creates a new branch and then switches to said new branch.
  • A new branch is an isolated working environment where code can be developed on one's local system.
  • These branches can be modified and eventually mergered with other branches or the main branch.
  • Before a branch is mergered to the main branch, a PR or pull request must be carried out. A PR is simply a peer review to make sure your code would be an effective addition to the overall application or website.
  • The main branch is a copy of the actually project that can be accesable to multiple developers through GitHub.
  • Using Git commands, we can initiate (or create) a repo within GitHub, add our code to the repo, and view the different versions of our code, which are each stored in their own unique commit.
  • Developers use repositories, aslo known as repos, as folders to store; files that contain code, files that contain information about code, or the version history for all the files in the repository.
  • GitHub Issues is a built in to do list to keep you and your team on track.
  • Github Co-pilot is an AI code prediction that can prove very useful. It encourages comment writing and uses those comments to predict your thought-process and potential coding outcomes.

JavaScript

  • JavaScript or JS is a high level dynamic programming language that allows the developer to integrate logic into a website, which inturn, increases functional capabilities for the end users
  • A variable is a placeholder for a value or piece of information. Each variable has a type, a name, and a piece of information stored within.
  • Variables may hold values that have different data types; strings, numbers, booleans, arrays, objects, etc
  • A string is a potential vaulue of a varible that stores text. A string is signified by single quotes.
  • A boolean is a variable that can only hold an output value of true or false.
  • An operator is a mathamatical symbol, and corresponding mathematical operation, that produces a rusult based on two values or variables.
  • An array is a structure that allows the developer to store multiple values in a single refrence. The information contained is usually similar.
  • Arrays are especially useful when we want to perform a task on multiple pieces of data.
  • An Index number is number that identifies the location of a piece of data within an array.
  • A for loop uses a pattern of indices to perform a task on all the items within an array by following a single block of code, to be executed over and over.
  • A for loop contains three parts; the intitial statement determines the starting point of our loop, the second statement is the condition to be met in order for the control flow to exit the loop, and the third statement is what allows the array to iterate over each item. (ex: x++)
  • Index numbers to begin at zero and proceed in numerical order.
  • A function is a segment or block of code that is easily accesable by calling that functions name.
  • A function must first be defined and then named.
  • Control flow is the order in which a computer executes code in a script. In JS the work flow is executed from top to bottom.
  • A conditional statement is a statement based on certain variable conditons. They allow for decision-making to be added to your code. If the conditions within the statement are met, the control flow is diverted to a correlating code block.
  • If statements, IF else statements, and else if statements are all conditional statements.
  • Each varaible condition within a conditional statemnt is treated as a boolean
  • A semi-colon at the end of a line indicates where a statement ends.
  • The .log() method will output whatever lies within the parenthesis to the console.
  • We use the console.log(); method to output a message on the console based on an added argument. The console.log method allows developers to easily tesst code.
  • Chrome Dev Tools are super useful for viewing console.log() outputs and potential rusults of developing code blocks.
  • Comments left within blocks of code are represented with /*...comments...*/. (or if line isn't broken //)
  • A database of comments, error codes, and solutions is useful as similar problems arrise in the future.
